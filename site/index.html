About the Ranking System

Our ranking system uses a sophisticated two-layer PageRank algorithm with enhanced EM convergence to generate completely unbiased, data-driven college football rankings. The system processes only authentic FBS game data to ensure accurate competitive analysis.

Key Features

  • FBS-Only Processing: Analyzes only authentic Division I FBS games for accurate competition assessment
  • Two-Layer PageRank: Separate team and conference ranking algorithms with mutual reinforcement
  • Conference Strength Adjustment: Relative scaling based on inter-conference performance
  • Bootstrap Confidence: Statistical uncertainty analysis for ranking reliability
  • Industry-Grade Validation: Comprehensive data integrity checks and edge condition handling
  • Bias Auditing: Advanced neutrality metrics ensure mathematical fairness

Automatic Updates

Rankings are automatically updated every Monday at 3:30 AM ET during the football season using the latest game results. The system:

  • Fetches fresh data from the College Football Data API
  • Runs complete data validation and integrity checks
  • Generates new rankings using the full mathematical pipeline
  • Publishes updated tables and JSON data automatically
  • Switches to final retrospective rankings after the season

Mathematical Methodology

The ranking algorithm follows a rigorous mathematical approach:

  • Graph Construction: Teams and conferences modeled as directed graphs with weighted edges
  • Edge Weighting: Victory margins adjusted for venue advantage and temporal decay
  • EM Convergence: Iterative algorithm converges to stable ratings with confidence bounds
  • Neutrality Optimization: Bias metric B minimization ensures conference-neutral results

Data Sources & Integrity

All data sourced from the official College Football Data API with comprehensive validation:

  • Schema validation with strict type checking
  • FBS classification verification against master lists
  • Duplicate detection and outlier analysis
  • Season consistency and game status validation
  • API reliability with exponential backoff and fallback mechanisms
